About PDFNet SDK

PDFNet SDK is a high-quality, industry-strength PDF library meeting requirements of the most demanding and diverse applications. Using PDFNet you can write stand-alone, cross-platform, and reliable commercial applications that can read, write, edit, print and display PDF documents. PDFNet SDK is available as a .NET component and as a cross-platform Java and C/C++ PDF library available on a wide range of platforms (i.e. Windows, Linux, Mac OS X, Solaris, Windows Mobile, iOS, Android etc). For more information, please visit us at www.pdftron.com.


Note about version 6.1.0

PDFNet for Android is now available in a new Standard feature reduced variant (a.k.a. 'lean'). The Standard library offers the same viewing, annotation, and editing capabilities of the fully featured library however much smaller. The parts that are missing are relatively rarely used components:

  • Convert, Optimizer, Redactor, Stamper, Flattener classes
  • PDF/A validation/conversion
  • Converting PDF pages to TIFF and PNG formats (ie, PDFDraw will not work when using these formats)

The Full variant, also found in the Lib folder, contains all the features above. Another change for this version is that embedded fonts, CMaps, and standard PDF resources were separated from the libraries into a file ('pdfnet.res') that can be shared among multiple .so libs. Check the Resource section for more information.

Starting with 6.1.0 the library only supports Android 2.3 (Gingerbread) or higher.


Package Directory Structure Overview

  • Doc - The folder containing PDFNet API Reference.
  • Headers - The folder containing C++ headers for developing native code.
  • Lib - The folder containing PDFNet libraries.
  • Resource - The folder containing the PDFNet resource file.
  • Samples - The folder containing sample projects and the Tools library.
  • Getting Started with PDFNet for Android.pdf - A tutorial with a step-by-step example for a basic viewer using PDFViewCtrl.
  • legal.txt - Legal and copyright information.
  • license.pdf - CPU license agreement and warranty for this product.
  • readme.html - This file.
  • version.txt - Native and SDK version information.

Headers

  • Contains the C++ header files that can be used with the native library in 'Lib' folder. These files are only provided for developing native code. Note that some unsuitable classes, including PDFView, PDFViewCtrl and Print, are not included.

Lib

  • Contains the PDFNet library (PDFNet.jar) for PDFViewCtrl class. The libraries support Android 2.3 or higher.
  • Contains the native Standard and Full libraries 'armeabi/libPDFNetC.so' (arm version), 'armeabi-v7a/libPDFNetC-v7a.so' (arm-v7 version), and 'x86/libPDFNetC.so' (x86 version). Note that the x86 library has only been tested on Android 4.1 (API 16).
  • To use the SDK you will need to compile with API level 11 (or higher) as the SDK uses some Honeycomb features if those are present at run-time.
  • Also note that it will be necessary to add to your res/raw folder the PDFNet resources file ("pdfnet.res"). This file contains information about embedded fonts, etc. Please, see Resource section for more information.

Resource

Contains the PDFNet resources file that contains information about embedded fonts, CMaps, and standard PDF resources.

This file was included to allow developers to have more control over the size of the generated APK. If you want to create an APK with two or more shared libraries (let's say, for arm and arm-v7) you need to include the resources only once.

To correctly view generic PDF documents you must load the external resources file using one of the following options:

  • PDFNet.initialize() includes an extra parameter that can be used to load the resource file from the application bundle. For example: PDFNet.initialize(this, R.raw.pdfnet). This method will copy the file from 'res/raw/pdfnet.res' to the private storage area of the app (Context.openFileOutput()). An exception will be thrown if there is no sufficient space to save the file, or if the resource ID can't be found. Please note that the resource file must be named "pdfnet.res" when using this approach.
  • PDFNet.initialize(Context) as before followed by PDFNet.setResourcesPath("path/to/resources/file"). With this method you are handling installation of the resource file on your own. For example, the resource file can be downloaded on demand and saved at any location. When the resource file is ready for use it can be loaded using PDFNet.setResourcesPath().

Samples

  • PDFViewCtrlDemo shows how to use PDFViewCtrl to easily build a PDF viewer with a rich set of functionalities.
    Some hints about using this viewer:
    1. pinch or double tap to zoom;
    2. long press in an empty region to bring up annotation menu;
    3. long press on text for text selection;
    4. single tap or long press on a annotation to select it;
    5. tap in the top quarter to bring up the top tool bar (for text se arch);
    6. tap in the bottom quarter to bring up the bottom tool bar (for page navigation);
    7. click the option menu on the system tool bar for more options.
  • PDFViewCtrlDemoTools library offers, via PDFViewCtrl's tool interface, UI modules that use PDFNet functions to add extra ready-to-use components such as annotation editing, text selection, text search, form filling, quick menu, etc. It is structured in a way that allows the developer to quickly customized it for a rapid development project. This project is shipped as an Android library.
  • PDFDrawDemo is a simple example that shows how PDFDraw can be used to make a simple PDF viewer. Tap top (bottom) half of the viewer to turn pages backward (forward).
  • MiscellaneousSamples is an application that shows the functionalities of the PDFNet SDK and includes the samples found in the sample code page. Inspect its source code to learn how to use some of the main classes and features of the SDK, or just run the sample and check the results.
    Notes:
    • Some samples are disabled due to the fact that the Standard libraries do not include some features. If using the Full libraries please look at the source code of such samples to make the appropriate changes.

How to Build with Eclipse

It is convenient to build and debug an Android project using Google's ADT Plugin. The following steps brief how to build, say PDFViewCtrlDemo and PDFViewCtrlTools, using Eclipse with ADT.

If you still do not have the IDE installed, we suggest using the ADT Bundle.

First we need to build the PDFViewCtrlTools library:

  • Copy the PDFNet.jar file from 'Lib' to the PDFViewCtrlTools's 'libs' folder. You only need this jar file to build the Tools library because this project will not be executed as an Android application.
  • In Eclipse, go to 'File -> New -> Project...' and select 'Android -> Android Project from Existing code'.
  • Select the PDFViewCtrlTools folder as the root directory and click 'Finish'.
  • Right click the project 'PDFViewCtrlTools' in the Package Explore panel and select 'Properties'.
  • Under the 'Android' category, make sure the 'Is Library' checkbox is checked and if a target API is selected.
  • You now are ready to build the library.

Now we will add the PDFViewCtrlDemo project into Eclipse:

  • You don't need to copy the PDFNet.jar again into this project, since we will referece the Tools project later. But you must add to your project the native library(ies). Copy the content from 'Lib\Standard' or 'Lib\Full' to the PDFViewCtrlDemo's 'libs' folder. Your project libs folder should have something like this structure:
    • libs\armeabi\libPDFNetC.so
    • libs\armeabi-v7a\libPDFNetC-v7a.so
    • libs\x86\libPDFNetC.so
    • libs\android-support-v4.jar
    You can copy all the subfolders ('armeabi', 'armeabi-v7a' and 'x86') to make the output apk compatible on all platforms. Android will load the proper library during installation. If you know which platform is targeted, you only need to copy one of them.
  • In Eclipse, go to 'File -> New -> Project...' and select 'Android -> Android Project from Existing code'.
  • Select the PDFViewCtrlDemo folder as the root directory and click 'Finish'.
  • Now we need to reference the Tools library. Right click the project 'PDFViewCtrlDemo' in the Package Explore panel and select 'Properties'.
  • Under the 'Android' category, in the Library section, click 'Add...'
  • The next dialog should show all the projects opened in Eclipse that are marked as 'Is Library'. Choose PDFViewCtrlTools and click 'OK'.
  • The Library section should show now the PDFViewCtrlTools in the list, and if everything is correct a green check will be shown. Please note that Eclipse will not work correctly if the library project being referenced is not placed in the same drive, since Eclipse/Android do not use the full path.
Now, you are ready to build and run the project given that you have an Android device available. You can use either a virtual device or a hardware device.

Using ProGuard

If you plan to use ProGuard on your APK before releasing your app, you will need to add to your proguard configuration file the following lines:
  • -keep class pdftron.PDF.PDFViewCtrl$RenderCallback { *; }
  • -keep class pdftron.Filters.CustomFilter$CustomFilterCallback { *; }
  • -keep class pdftron.PDF.ProgressMonitor { *; }
  • -keep class pdftron.SDF.ProgressMonitor { *; }
This will ensure that the native calls to the rendering callback will not be obfuscated.
