About PDFNet SDK

PDFNet SDK is a high-quality, industry-strength PDF library meeting requirements of the most demanding and diverse applications. Using PDFNet you can write stand-alone, cross-platform, and reliable commercial applications that can read, write, edit, print and display PDF documents. PDFNet SDK is available as a .NET component and as a cross-platform Java and C/C++ PDF library available on a wide range of platforms (i.e. Windows, Linux, Mac OS X, Solaris, Windows Mobile, iOS, Android etc). For more information, please visit us at www.pdftron.com.

Package Directory Structure Overview

  • Doc - The folder containing PDFNet API Reference.
  • Lib - The folder containing PDFNet libraries.
  • Headers - The folder containing C++ headers for developing native code.
  • Samples - The folder containing some sample projects.
  • legal.txt - Legal and copyright information.
  • version.txt - Native and SDK version information.
  • license.pdf - CPU license agreement and warranty for this product.
  • readme.html- This file.

Lib

  • Contains the PDFNet library (PDFNet.jar) and the add-on tool library (Tools.jar) for PDFViewCtrl class. The libraries support Android 2.2 or higher.
  • Contains the native libraries 'armeabi/libPDFNetC.so' (arm v6 version), 'armeabi-v7a/libPDFNetC-v7a.so' (arm v7 version), and 'x86/libPDFNetC.so' (x86 version). Note that the x86 library has only been tested on Android 4.1 (API 16).
  • Tools.jar library offers, via PDFViewCtrl's tool interface, UI modules that use PDFNet functions to add extra ready-to-use components such as annotation editing, text selection, text search, form filling, quick menu, etc. The source code of Tools.jar is available in the licensed version and customers are able to fully customize it.
  • To use the SDK you will need to compile with API level 11 (or higher) as the SDK uses some Honeycomb features if those are present at run-time.

Headers

  • Contains the C++ header files that can be used with the native library in 'Lib' folder. These files are only provided for developing native code. Note that some unsuitable classes, including PDFView, PDFViewCtrl and Print, are not included.

Samples

  • Contains samples for certain PDFNet classes. The 'PreBuilt' sub-directory contains the built APKs of the samples on different architectures.
  • PDFViewCtrlDemo shows how to use PDFViewCtrl to easily build a PDF viewer with a rich set of functionalities.

    Some hints about using this viewer:
    (1) pinch or double tap to zoom;
    (2) long press in an empty region to bring up annotation menu;
    (3) long press on text for text selection;
    (4) single tap or long press on a annotation to select it;
    (5) tap in the top quarter to bring up the top tool bar (for text se arch);
    (6) tap in the bottom quarter to bring up the bottom tool bar (for page navigation);
    (7) click the option menu on the system tool bar for more options.

  • PDFDrawDemo is a simple example that shows how PDFDraw can be used to make a simple PDF viewer. Tap top (bottom) half of the viewer to turn pages backward (forward).
  • PDFNetSDKSampleDemo is an application that shows the functionalities of the PDFNet SDK and includes the samples found in the sample code page. Inspect its source code to learn how to use some of the main classes and features of the SDK, or just run the sample and check the results.

    Notes:
    (1) To run the DigitalSignatures sample you will need to donwload and install separately the Spongy Castle libraries. Place the jar files in the 'libs' folder of the project and change DigitalSignaturesSample.java and MySignatureHandler.java files accordingly.
    (2) The application is optimized to run on tablets, and will not run on handset devices.

How to Build with Eclipse

It is convenient to build and debug an Android project using Google's ADT Plugin. The following steps brief how to build, say PDFViewCtrlDemo, using Eclipse with ADT.
  • Setup ADT in Eclipse as described in this article;
  • Copy the content from 'Lib' to the sample's 'libs' folder. Note that you can copy all the subfolders ('armeabi', 'armeabi-v7a' and 'x86') to make the output apk compatible on all platforms. Android will load the proper library during installation. If you know which platform is targeted, you only need to copy one of them.
  • Go to “File -> New -> Projects…” and select “Android -> Android Project…”;
  • Type in the project name (say PDFViewCtrlDemo), and select “Create project from existing source” and in “Location :”, browse to '/PDFViewCtrlDemo' foler;
  • Click “Next” to select the Build Target, say Android 3.0, and then click “Finish”. Note that PDFNet Android SDK supports Android 2.2 and higher;
  • Right click the project “PDFViewCtrlDemo” in the Package Explore panel and select “Properties”;
  • Select “Java Build Path -> Libraries”;
  • Click “Add Jars…” and browse to “PDFViewCtrlDemo/libs/” and select PDFNet.jar. You will do this again to select Tools.jar;
Now, you are ready to build and run the project given that you have an Android device available. You can use either a virtual device or a hardware device.

Using ProGuard

If you plan to use ProGuard on your APK before releasing your app, you will need to add to your proguard configuration file the following line:
  • -keep class pdftron.PDF.PDFViewCtrl$RenderCallback { *; }
This will ensure that the native calls to the rendering callback will not be obfuscated.
